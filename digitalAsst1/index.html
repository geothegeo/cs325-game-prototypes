Block Snake

  • Originality: It's a classic Snake game with a randomly spawning obstacle, and you can move past the borders.
  • Technical Merit: The visual and audio elements are on point. The random spawning "blocks" are pretty interesting.
  • Prototype Postmortem: I learned a lot of the basics in dealing with Phaser 3. There was a lot to learn and consider when giving each element in the game a class and properties. I also gained insight into adding images, music, movement, randomness, and object collision. The hardest part by far was probably figuring out how the objects should interact with each other.
  • Prototype Assets:
    • Snake body: Black Square PNG
    • Food: Apple PNG
    • Obstacle: Brick Cliparts #37174
    • Background Music: "Happy Dreams" MP3
    • Biting Sound Effect: "Bite into apple 1" MP3
    • Dying Sound Effect: "Retro 8-bit arcade style game sound, negative, death, die or lose life 3" MP3
  • Prototype Closest Other Game: This game is close to the classic Nokia Snake game. The codebase I used is from here.
  • High Concept: Collect apples and grow in size until there are no more apples to collect.
  • Theme/Challenge: The challenge I tackled: "Things must only be added as the game progresses, not removed".
  • Mandated Variety: State which input, randomness, genre, and play style you used in this prototype.
    • Input: keyboard
    • Randomness: some randomness
    • Genre: Family, Puzzle
    • Play Style: Collector
  • Prototype Goal: I think the game mechanic this game is going for is "Movement".
  • Player Experience Goals: I want the player to have fun and be on their toes as the game gets increasingly harder.
  • Gameplay: The player can only use arrow keys to move: left (left arrow), right (right arrow), up (up arrow), and down (down arrow). The player will need to try to eat as many apples as possible while avoiding the random bricks. The catch is the more apples the player eats, the faster the snake will move, so the person must be very quick-witted when moving around on the board.
  • Strategies: There is no real strategy to this game, but the player does need to have good dexterity and reaction times.
  • Story/Setting/Premise: There is really not any story or setting that made the game what it is. Maybe the snake got hungry and he needed to eat apples, but in reality, life is not easy and his body is cursed to grow very fast as he ate.
  • Target Audience: Anyone who wants to play a short arcade game.
  • Play Time: It would probably take about 10 min to finish the game if you are lucky.
